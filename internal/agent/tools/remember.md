Store a durable fact in long-term memory so it is available in future sessions.

Use this the moment you learn something reusable about this project or the user that would save work next time: a build/test/lint command, a project convention or architecture decision, where an important thing lives, an environment quirk, or a stated user preference. Capture sparingly and specifically — one self-contained sentence per fact.

Do NOT store transient task state ("currently editing foo.go"), secrets, or anything that only matters for the current task. Near-duplicate facts are merged automatically, so restating a known fact is harmless.

Relevant memories are also surfaced to you automatically at the start of a turn; this tool is for deliberately recording something you just learned.
